This project is **experimental and evolving**. Contributions are welcome, but please note that some areas of the codebase may be rough or AI-assisted. Use your best judgment, and feel free to propose changes to this document in a pull request.

## How Can I Contribute?

### Reporting Bugs

* Check if the bug is already reported in the [Issues](https://github.com/FitimZulfiju/Ai-CV-application-generator/issues).
* If not, create a new issue with as much detail as possible (OS, browser, LLM used, steps to reproduce, error messages).

### Suggesting Enhancements

* Open an [Issue](https://github.com/FitimZulfiju/Ai-CV-application-generator/issues) with the tag `enhancement`.
* Explain the rationale and the problem it solves.

### Pull Requests

1. Fork the repo and create a branch from `main`.
2. If your code includes new features or fixes, add tests where possible.
3. Ensure the test suite passes.
4. Follow existing code style and project conventions.
5. Submit a pull request describing your changes and reasoning.

## Style Guide

### Git Commit Messages

* Use [Conventional Commits](https://www.conventionalcommits.org/en/v1.0.0/) format.
* Examples:

  * `feat: add support for local LLMs`
  * `fix: resolve PDF export padding`

### Coding Standards

* Follow standard C# and .NET coding conventions.
* Use MudBlazor components consistently with the existing UI design.
* When adding new code, consider maintainability and readability.

## Community & Feedback

* Join our [GitHub Discussions](https://github.com/FitimZulfiju/Web-CV-application-generator/discussions) for general discussion.
* Open a [GitHub Issue](https://github.com/FitimZulfiju/Web-CV-application-generator/issues) for specific architectural questions or feedback.
* Contributions, suggestions, and constructive criticism are welcome, especially around structure, AI integration, and usability.
