The following is a set of guidelines for contributing to **AiCV**. These are mostly guidelines, not rules. Use your best judgment, and feel free to propose changes to this document in a pull request.

## How Can I Contribute?

### Reporting Bugs

* Check if the bug is already reported in the [Issues](https://github.com/FitimZulfiju/Ai-CV-application-generator/issues).
* If not, create a new issue. include as much detail as possible (OS, Browser, LLM used, error messages).

### Suggesting Enhancements

* Open an [Issue](https://github.com/FitimZulfiju/Ai-CV-application-generator/issues) with the tag `enhancement`.
* Explain the "Why" behind the feature.

### Pull Requests

1. Fork the repo and create your branch from `master`.
2. If you've added code that should be tested, add tests.
3. Ensure the test suite passes.
4. Make sure your code lints.
5. Issue that pull request!

## Styleguide

### Git Commit Messages

* Use the [Conventional Commits](https://www.conventionalcommits.org/en/v1.0.0/) format.
* Example: `feat: add support for local LLMs` or `fix: resolve PDF export padding`.

### Coding Standards

* Follow standard C# and .NET coding conventions.
* Use MudBlazor components for UI consistent with the existing design.

## Community

Join our discussions if available or reach out via email for specific architectural questions.
